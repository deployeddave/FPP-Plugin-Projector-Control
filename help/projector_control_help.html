FPP Projector Control Plugin

Complete setup guide for controlling projectors via RS232 serial, TCP/IP network, and PJLink from Falcon Player.

Overview

The Projector Control plugin allows Falcon Player (FPP) to automatically power on, power off, switch inputs, mute, freeze, and perform other control operations on a connected projector — synchronized with your light show playlists.

When you select a projector and save your settings, the plugin generates shell scripts for every command your projector supports. Those scripts appear as FPP Commands and can be attached to playlist events, triggers, or scheduled actions — giving you fully automated projector control without touching the remote.

What the plugin can do

  • Power the projector on and off on a schedule
  • Switch input sources (HDMI1, HDMI2, VGA, etc.)
  • Blank/mute the image mid-show without powering down
  • Freeze the image
  • Mute audio
  • Any other command your projector supports via RS232

Supported connection types

Serial / RS232

USB-to-serial adapter plugged into the Pi, connected to the projector's DB9 RS232 port. Most common setup. Works with nearly all projectors.

TCP/IP Network

Sends RS232-equivalent commands over a network connection. Projector must be connected to your LAN and have network control enabled.

PJLink

Open industry standard for IP-based projector control. Uses Perl's Net::PJLink library. Requires a PJLink-compatible projector on your LAN.

Telnet (TCP port 23)

A subset of TCP mode. Some projectors (e.g., Dukane) accept RS232-equivalent ASCII commands over a Telnet connection on port 23.

Requirements

For Serial (RS232) control

  • A projector with a DB9 RS232 serial control port (check your projector's manual — most mid-range and above projectors have one)
  • A USB-to-serial adapter with an FTDI FT232R or FT232RL chip (strongly recommended over Prolific PL2303 or CH340 chips for reliability)
  • A DB9 straight-through serial cable (female-to-female is typical)
  • Possibly a null-modem adapter — required for some projectors if commands are received but ignored
⚠ USB Adapter Chip Matters FTDI-chip adapters (FT232R/RL) are the most reliable on Raspberry Pi. Prolific PL2303 and CH340 chips are cheap and common but can cause hangs or intermittent failures. If your serial connection is unreliable, the chip is the first thing to suspect.

For Network (TCP/IP or PJLink) control

  • Projector connected to your local network via ethernet or Wi-Fi
  • Network control enabled in the projector's OSD menu (varies by manufacturer)
  • Projector's IP address (static IP recommended — DHCP can change it)
  • For PJLink: the Net::PJLink Perl library (installed automatically by fpp_install.sh)
  • For TCP: the projector's control port number (check your projector's manual — commonly 4352, 3629, 7142, or 23)

General requirements

  • FPP version 2.0 or higher (this fork supports FPP 2.x through 10.x)
  • PHP installed on the FPP device (included with FPP by default)
  • Internet access during initial install (to download Net::PJLink via CPAN)

Installation

Install the plugin through the FPP Plugin Manager, or manually from the GitHub repository.

Method 1 — FPP Plugin Manager (recommended)

Open the Plugin Manager In the FPP web UI, go to Content Setup → Plugin Manager.
Find and install Projector Control Search for "Projector Control" in the available plugins list and click Install.
Wait for install to complete The install script runs fpp_install.sh which installs the Net::PJLink Perl library via CPAN. This may take 2–5 minutes on first install.
Reboot when prompted The installer sets a restart flag. Reboot FPP to complete the setup.

Method 2 — Manual install from this fork

If you want to use this updated fork (with additional projectors and bug fixes) instead of the original:

# SSH into your FPP device, then:
cd /home/fpp/media/plugins
git clone https://github.com/deployeddave/FPP-Plugin-Projector-Control.git FPP-Plugin-Projector-Control
cd FPP-Plugin-Projector-Control
bash fpp_install.sh
ℹ What fpp_install.sh does It calls FPP's built-in plugin update script, sets serial port permissions (chmod a+w /dev/tty*), installs Net::PJLink via CPAN, and sets a reboot flag.

After rebooting, navigate to Content Setup → Plugin Manager and you should see "Projector Control" listed as installed. Click its settings link to configure it.

Serial / RS232 Setup

This is the most common setup. The Raspberry Pi does not have a native RS232 port, so you use a USB-to-serial adapter to bridge between the Pi's USB port and the projector's DB9 serial port.

Step 1 — Connect the hardware

🍓
Raspberry Pi — any USB port
↓ USB cable
🔌
USB-to-Serial Adapter (FTDI chip recommended) — appears as /dev/ttyUSB0
↓ DB9 straight-through cable (female-to-female)
📽
Projector RS232 port (DB9 male on most projectors)
⚠ Null Modem Adapter Some projectors require a null-modem (crossover) adapter between the cable and the projector. If the Pi can see the device but the projector never responds, try adding a null-modem adapter. This crosses the TX/RX wires and is required when both ends expect to be the "DTE" device.

Step 2 — Verify the adapter is detected

SSH into your FPP device and run:

ls /dev/ttyUSB*

You should see /dev/ttyUSB0 (or ttyUSB1, etc. if you have other devices). If nothing appears, the adapter is not being recognized — check the USB connection and try a different port.

To see more detail about which chip was detected:

dmesg | grep -i "usb\|serial\|tty" | tail -20

Look for lines mentioning FTDI, ftdi_sio, pl2303, or ch341 — this tells you which driver loaded and confirms the adapter was recognized.

Step 3 — Check permissions

The plugin install script runs chmod a+w /dev/tty* to ensure FPP can write to the serial port. If you still get permission errors, run manually:

sudo chmod a+rw /dev/ttyUSB0
⚠ Device name can change on reboot If you have multiple USB devices, /dev/ttyUSB0 may become /dev/ttyUSB1 after a reboot depending on plug-in order. For a permanent fix, use the stable device path instead — see Stable Device Names below.

Stable device names (advanced)

Instead of /dev/ttyUSB0, use the persistent symlink that Linux creates based on the adapter's hardware ID:

ls /dev/serial/by-id/

You'll see something like:

usb-FTDI_FT232R_USB_UART_A50285BI-if00-port0

In the plugin's Device dropdown, you won't see this path directly — but you can create a udev rule to always map your specific adapter to a fixed name like /dev/projector. This is optional but eliminates the device-numbering problem entirely.

Network (TCP/IP) Setup

Some projectors accept RS232-equivalent commands over a network connection. This eliminates the need for a physical serial cable.

Enable network control on the projector

This varies by manufacturer but the process is similar across brands:

  1. Open the projector's OSD menu (Menu button on remote)
  2. Navigate to Settings → Network or Advanced → LAN Control
  3. Enable the network control or RS232-over-LAN option
  4. Note the projector's IP address and control port number
  5. Set a static IP on the projector (highly recommended)
ℹ Common control port numbers by manufacturer
ManufacturerTypical Port
NEC7142
Dukane (Telnet)23
ViewSonic4661
Epson3629
Optoma (older)4352
BenQ (LAN)8000
Always verify with your specific projector's manual.

Plugin settings for TCP mode

When you select a TCP-protocol projector in the plugin, the UI will show an IP address field and a Port field. Enter the projector's LAN IP and the control port from its manual.

⚠ Standby power mode matters Many projectors cut network power in standby mode, which means the Power On command may not work over the network. Check your projector's OSD for a "Standby LAN Control" or "Network in Standby" option and enable it.

PJLink Setup

PJLink is an open, manufacturer-neutral standard for controlling projectors over a network. It is supported by most modern business projectors and provides a consistent command set regardless of brand.

Verify your projector supports PJLink

Check the projector's specifications or manual for "PJLink Class 1" or "PJLink Class 2" support. If listed, the projector can be controlled via this plugin's PJLINK protocol option.

Enable PJLink on the projector

  1. Open the projector OSD → Network/LAN settings
  2. Enable PJLink (may be labelled "Network Control" or "AMX/Crestron/PJLink")
  3. Set a PJLink password if required (or leave blank if not used)
  4. Note the projector's IP address

Plugin settings for PJLink mode

Select the PJLINK entry from the projector dropdown. The UI will show only an IP address field (PJLink always uses port 4352 — no port field needed). If your projector has a PJLink password set, enter it in the Password field.

ℹ PJLink uses a Perl library PJLink control is handled by the pjlinkutil.pl Perl script which requires the Net::PJLink CPAN library. This is installed automatically during plugin install. If PJLink commands fail, verify the library is present by running perl -e "use Net::PJLink" — no output means it's installed.

Cable Guide

DB9 connector pin reference

PinSignalDirection (from projector)
1CD (Carrier Detect)Input
2RXD (Receive Data)Input — projector receives commands here
3TXD (Transmit Data)Output — projector sends responses here
4DTROutput
5GND (Signal Ground)Ground
6DSRInput
7RTSOutput
8CTSInput
9RI (Ring Indicator)Input

Only pins 2, 3, and 5 are required for basic control. Many USB-to-serial adapters only wire these three pins.

Cable types explained

Straight-through cable

Pin 2 → Pin 2, Pin 3 → Pin 3. Used when one end is a DTE device (PC/Pi adapter) and the other is a DCE device (projector). This is the most common requirement.

Null modem / crossover cable

Pin 2 → Pin 3, Pin 3 → Pin 2. Crosses TX and RX. Required when both devices are DTE. Some projectors need this — if straight-through doesn't work, try adding a null-modem adapter.

✓ Quick test for cable type If the projector's RS232 indicator light flashes when you send a command (visible on some models) but the projector doesn't respond, your cable type is likely correct but baud rate or command format is wrong. If there's no indicator activity at all, try swapping cable type.

How Scripts Work

Each command key in your projector's entry (ON, OFF, HDMI1, AV_MUTE_ON, etc.) generates two files:

Script locations

/home/fpp/media/scripts/PROJECTOR-ON.sh
/home/fpp/media/plugins/FPP-Plugin-Projector-Control/commands/PROJECTOR-ON.sh

The scripts directory copy is what FPP's script runner executes. The commands directory copy is what gets listed in descriptions.json for the FPP Command UI. Both are always kept in sync.

What a generated script contains

#!/bin/sh

#Created by FPP-Plugin-Projector-Control
#
/usr/bin/php /home/fpp/media/plugins/FPP-Plugin-Projector-Control/proj.php -dSERIAL -s/dev/ttyUSB0 -cON

The script calls proj.php with three arguments:

ArgumentMeaning
-d SERIAL or -d IPConnection type (SERIAL, IP, or PJLINK)
-s /dev/ttyUSB0Serial device path (serial mode only)
-c ONThe command key to execute (ON, OFF, HDMI1, etc.)

What proj.php does when a script runs

  1. Checks that a projector is configured and the plugin is enabled — exits silently if not
  2. Looks up the selected projector in projectorCommands.inc and finds the command key
  3. Retrieves the raw hex bytes for that command
  4. Opens the serial port, sends the bytes, waits 1 second, reads any response, closes the port
  5. Logs everything to the plugin log file

Command naming in FPP

In FPP's Command interface, each script appears as:

"Projector Control-Projector ON"
"Projector Control-Projector OFF"
"Projector Control-Projector HDMI1"
"Projector Control-Projector AV_MUTE_ON"

The display name is always Projector Control-Projector followed by the command key name exactly as it appears in projectorCommands.inc. The key name is the display name — no other configuration controls this.

Using Commands in FPP

Once the plugin is configured and saved, projector commands appear automatically in several places within FPP.

Method 1 — FPP Command Presets

Open Command Presets Go to Content Setup → Command Presets in the FPP UI.
Create a new preset Click Add Preset. In the Command dropdown, select Run Script. In the Script field, choose a PROJECTOR-*.sh script.
Name and save it Give it a descriptive name like "Projector ON" and save. It's now available anywhere FPP accepts commands.

Method 2 — Playlist Events

In a playlist, add a Script item at the appropriate position. Select the projector script from the dropdown. The script runs at that point in the playlist automatically.

Typical playlist structure for a show with a projector:

Playlist PositionAction
Lead-in (before show)Run PROJECTOR-ON.sh — power up and allow warm-up time
Start of showRun PROJECTOR-HDMI1.sh — switch to show input
Between songs (optional)Run PROJECTOR-AV_MUTE_ON.sh — blank image during quiet transitions
End of showRun PROJECTOR-OFF.sh — power down gracefully

Method 3 — FPP Triggers / GPIO

Projector commands can be attached to GPIO input triggers. For example, a physical button wired to a Pi GPIO pin can fire a command preset that triggers PROJECTOR-ON.sh.

Method 4 — Scheduler

Use FPP's built-in scheduler to run a playlist containing a projector ON or OFF command at a specific time each day. This is the simplest way to automate a fixed show schedule.

⚠ Allow warm-up time for the projector Most lamp-based projectors take 30–90 seconds from Power ON command to a usable image. Add a delay sequence or silent lead-in after the power on command before your show content starts. Laser projectors are typically ready in under 10 seconds.

Show Integration Best Practices

Recommended show timing structure

Based on community experience from outdoor light show operators:

TimeActionNotes
T-2:00Power ONAllows lamp to warm up fully before show
T-1:00Switch to show inputConfirm correct source is active
T-0:00Show startsLights + projector in sync
Show endPower OFFAllow projector to cool down gracefully

Graceful power-off is important

Projectors (especially lamp-based) need time to cool the lamp before power is cut. The Power OFF RS232 command starts the projector's own cool-down fan cycle. Never cut power to the projector outlet without sending the OFF command first — it degrades lamp life and can damage the projector.

✓ Auto power-off as a safety net Enable the projector's built-in Auto Power Off feature (in its OSD menu) set to 15–30 minutes of no signal. This acts as a backup in case the plugin's OFF command fails for any reason — the projector will shut itself down rather than run all night.

Using AV Mute / Blank during a show

For outdoor shows with multiple song segments, blanking the projector image between segments (using AV_MUTE_ON) is more graceful than flickering content. The image disappears instantly and reappears instantly, without the thermal stress of power cycling. This works on projectors that have an AV Mute or Blank command defined in their entry.

Running projector control from a separate Pi

In some show setups (particularly where the projector is mounted at the far end of a long HDMI cable), a dedicated Raspberry Pi Zero W running FPP Remote is placed physically near the projector. This Pi handles only the RS232 serial control via a USB-to-serial adapter plugged into the projector. The main FPP instance sends the projector commands to the remote Pi via FPP's network command system.

Benefits of this approach: shorter serial cable run (the Pi Zero sits right next to the projector), keeps the main Pi clean, and allows the remote Pi to also handle local video output to the projector if needed.

Adding Your Projector

If your projector is not in the built-in list, you can add it by editing projectorCommands.inc.

Step 1 — Find your projector's RS232 command table

Search for: "[Your brand] [Your model] RS232 command table" or "[brand] [model] serial control guide PDF". Manufacturers publish these as PDFs on their support sites. Look for:

  • Power ON and Power OFF hex codes
  • Input source selection codes (HDMI, VGA, etc.)
  • Baud rate, data bits, parity, stop bits settings

Step 2 — Understand the hex format

RS232 commands are raw bytes. In PHP strings, each byte is written as \xNN where NN is the hex value. For example, hex byte 7E becomes \x7E in the PHP string.

If your projector's manual shows a command as: 7E 30 30 30 30 20 31 0D

That becomes in PHP: "\x7E\x30\x30\x30\x30\x20\x31\x0D"

Step 3 — Edit projectorCommands.inc

The file is located at:

/home/fpp/media/plugins/FPP-Plugin-Projector-Control/projectorCommands.inc

Add a new Array() block immediately before the final Array("NAME" => "-- Select Projector --", line. Use this template:

Array("NAME" => "Your_Projector_Model",
    "ON"        => "\x7E\x30\x30\x30\x30\x20\x31\x0D",  // ~0000 1 = Power ON
    "OFF"       => "\x7E\x30\x30\x30\x30\x20\x30\x0D",  // ~0000 0 = Power OFF
    "HDMI1"     => "\x7E\x30\x30\x31\x32\x20\x31\x0D",  // ~0012 1 = HDMI input
    "VGA"       => "\x7E\x30\x30\x31\x32\x20\x35\x0D",  // ~0012 5 = VGA input
    "BAUD_RATE" => "9600",
    "CHAR_BITS" => "8",
    "STOP_BITS" => "1",
    "PARITY"    => "none"
),
ℹ Key naming matters The key name (ON, OFF, HDMI1, etc.) becomes the command script name and the display name in FPP. Keep key names short and uppercase. Avoid spaces — use underscores instead (e.g., AV_MUTE_ON). Keys named NAME, BAUD_RATE, CHAR_BITS, STOP_BITS, and PARITY are reserved — do not use them as command keys.

Step 4 — Save and regenerate scripts

After editing the file, go back to the plugin settings page in FPP and hit Save again (even if no settings changed) to trigger script regeneration with your new projector entry.

Adding a network (TCP) projector

Array("NAME" => "MyProjector_Network",
    "PROTOCOL" => "TCP",
    "ON"       => "\x02\x00\x00\x00\x00\x02",
    "OFF"      => "\x02\x01\x00\x00\x00\x03",
    "IP"       => "",       // leave blank — entered in UI
    "PASSWORD" => "",
    "PORT"     => "7142"    // your projector's control port
),

ASCII-protocol projectors (e.g., BenQ, Epson)

Some projectors use plain ASCII text commands instead of binary hex. These still go into PHP strings the same way — you just write the ASCII character codes. For example, BenQ's *pow=on# with CR wrapping becomes:

"ON" => "\x0D\x2A\x70\x6F\x77\x3D\x6F\x6E\x23\x0D",  // \r*pow=on#\r

Or, since PHP interprets escape sequences in double-quoted strings, you can write it more readably using the \r and actual ASCII characters:

"ON" => "\r*pow=on#\r",

Both forms are equivalent in PHP.

Testing Commands from the Command Line

Before relying on the plugin during a show, test your serial connection directly from the Pi's command line. This isolates whether the problem is hardware/cabling or plugin configuration.

Method 1 — Run the proj.php script directly

cd /home/fpp/media/plugins/FPP-Plugin-Projector-Control
/usr/bin/php proj.php -dSERIAL -s/dev/ttyUSB0 -cON

This runs exactly the same code the generated shell scripts run. Check the log file immediately after for results.

Method 2 — Run the generated shell script

bash /home/fpp/media/scripts/PROJECTOR-ON.sh

Method 3 — Test with stty and echo (raw serial test)

This bypasses the plugin entirely and sends raw bytes directly to the serial port — useful for confirming your cable and projector are working at all:

# Set port parameters first
stty -F /dev/ttyUSB0 9600 raw -echo -echoe -echok

# Send a power ON command (Optoma ~0000 1 example)
printf '\x7E\x30\x30\x30\x30\x20\x31\x0D' > /dev/ttyUSB0

For ASCII-protocol projectors like BenQ:

stty -F /dev/ttyUSB0 9600 raw -echo
printf '\r*pow=on#\r' > /dev/ttyUSB0

Method 4 — Test PJLink directly

perl /home/fpp/media/plugins/FPP-Plugin-Projector-Control/pjlinkutil.pl \
  192.168.1.100 set_power_state 1
ℹ Listen for a response Most projectors send a response byte when they receive a command. To see it, open a second SSH session and run cat /dev/ttyUSB0 before sending the command in the first session. A response confirms two-way communication is working.

Troubleshooting

Projector commands do nothing and log shows "Plugin is DISABLED"

The Enable Plugin checkbox is set to OFF. Go to plugin settings and enable it, then save.

Projector commands do nothing and log shows "No Projector configured"

No projector has been selected, or the selection wasn't saved. Go to plugin settings, select your projector from the dropdown, and click Save.

Log shows "projector command not found: exiting"

The command key being requested doesn't exist in your projector's entry. This usually means the script was generated for a different projector and the current projector doesn't have that command key. Re-save the plugin settings with the correct projector selected.

Serial device not appearing in the dropdown

The USB-to-serial adapter isn't being detected. Check that it's plugged in and run ls /dev/ttyUSB* via SSH. If nothing appears, try a different USB port or a different adapter (FTDI-chip adapters are most reliable on Pi).

Device shows in dropdown but projector doesn't respond

Work through this checklist in order:

  1. Verify baud rate matches what your projector's manual specifies (check the projector's OSD too)
  2. Try swapping between a straight-through cable and a null-modem adapter
  3. Confirm the projector's RS232 port is enabled in the OSD (some projectors have this as a toggle)
  4. Test with raw stty/printf commands to rule out plugin issues
  5. Try the dSERIAL direct test and watch the log for what bytes were sent
  6. Verify your projector is in the correct power state — many projectors ignore RS232 when fully powered off (no standby power)

Commands work manually but not in a playlist

Check that:

  • The plugin is enabled
  • The script file has execute permissions (chmod +x PROJECTOR-ON.sh)
  • The serial device name hasn't changed since the scripts were generated (reboot can change ttyUSB0 to ttyUSB1 if you added another USB device)
  • You're not sending commands too quickly — the plugin waits 1 second after sending before closing the port; add a small delay between back-to-back commands in your playlist

PJLink commands fail with a Perl error

The Net::PJLink library may not be installed. Run:

perl -e "use Net::PJLink; print 'OK\n'"

If you get a "Can't locate" error, reinstall it:

PERL_MM_USE_DEFAULT=1 sudo cpan install Net::PJLink

Network projector can be pinged but commands don't work

  • Confirm network control is enabled in the projector's OSD
  • Verify the port number is correct for your model
  • Check that the projector's standby power mode allows network communication — enable "Network in Standby" or "Active Standby" in the OSD if available
  • Try pinging the projector from the Pi: ping 192.168.1.x
  • Try connecting manually: nc 192.168.1.x PORT — if it connects, the port is open

The wrong projector's commands are being sent

The scripts are generated at save time based on whatever projector was selected. If you've changed the projector since the last save, the old scripts are still in place. Go to plugin settings and click Save again to regenerate with the current selection.

Frequently Asked Questions

My projector isn't in the list. Can I still use the plugin?

Yes. Edit projectorCommands.inc and add your projector's entry following the template in the Adding Your Projector section. Your projector's RS232 command table should be in its manual or downloadable from the manufacturer's support site. Search for "[brand] [model] RS232 command table".

Do I need a null-modem adapter?

It depends on the projector. Start with a straight-through cable. If the projector doesn't respond but you can confirm the adapter is recognized by the Pi, try adding a null-modem adapter. This is often the fix for older projectors.

Can I control two projectors?

The plugin manages one projector at a time. For two projectors, you have two options: (1) use two separate FPP instances (one main, one remote), each with the plugin configured for their own projector; or (2) manually write custom scripts that call proj.php with different settings for each projector and place those scripts in the FPP scripts directory.

What's the difference between AV Mute and Power Off?

Power Off shuts down the projector — the lamp cools, the fan runs for 60–90 seconds, and startup from cold takes 30–90 seconds. AV Mute (or Blank) simply blacks the image instantly while the projector stays warm and on. For transitions during a show, AV Mute is almost always the right choice. Power Off is for end-of-night.

Will the plugin work if FPP is running as a remote (not master)?

Yes. The projector plugin runs locally on whichever FPP instance has it installed. If the projector is physically connected to a remote Pi, install the plugin on that remote Pi and configure it there. Playlist events on the master can trigger scripts on remotes using FPP's Remote Script command.

The projector powers on but then shuts down a few minutes into the show

Check the projector's OSD for Auto Power Off settings. Many projectors auto-shut-off after a set period with no signal or user input. Either disable Auto Power Off in the OSD or ensure your show is continuously providing a video signal.

Can I use this with a projector that has only USB, not RS232?

No. The plugin requires a proper RS232 control port. USB ports on projectors are typically for firmware updates, thumb drive media, or USB-to-HDMI display connections — they don't accept RS232 control commands. Check if your projector has a network control option and use the TCP mode instead.

The plugin generates scripts when I save, but I don't see them in playlists

Refresh the FPP UI — sometimes it caches the available scripts list. Also check that the /home/fpp/media/scripts/PROJECTOR-*.sh files actually exist with execute permission. Run ls -la /home/fpp/media/scripts/PROJECTOR* to verify.
